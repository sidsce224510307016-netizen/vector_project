Section 01

Problem Statement

Build a linear regression model to predict house prices from area, using Gradient Descent to minimise the Mean Squared Error (MSE) loss function.

🏠

Real-World Problem

Predict house prices (₹ Lakhs) based on property area (sq ft) — a classic supervised learning regression task.

📉

Core Challenge

Given a loss function L(w, b), find the weight w and bias b that minimise L — efficiently, without brute-force search.

∇

Vector Calculus Angle

The gradient ∇L = [∂L/∂w, ∂L/∂b] points in the direction of steepest increase. Moving opposite to ∇L descends to the minimum.

📊

Dataset

12 data points: house areas ranging 650–4500 sq ft with prices 45–300 ₹ Lakhs. Features normalised using z-score.

Section 02

Mathematical Model

Full formulation using vector calculus — hypothesis, loss function, gradient computation, and update rules.

Hypothesis

Linear Model

ŷ = w · x + b

w = weight (slope), b = bias (intercept), x = normalised area. Output ŷ is predicted price.

Loss Function

Mean Squared Error

L(w,b) = (1/2m) Σ (ŷᵢ − yᵢ)²

Smooth convex scalar field over parameter space. The factor ½ simplifies gradient computation.

Gradient

Partial Derivatives

∂L/∂w = (1/m) Σ (ŷ−y)·x
∂L/∂b = (1/m) Σ (ŷ−y)

The gradient vector ∇L = [∂L/∂w, ∂L/∂b] indicates the steepest ascent direction.

Update Rule

Gradient Descent Step

w ← w − α · (∂L/∂w)
b ← b − α · (∂L/∂b)

α = learning rate (0.10). Each step moves parameters in the direction of steepest descent.

Normalisation

Z-Score Scaling

x_norm = (x − μ) / σ

Prevents gradient instability. Ensures uniform gradient magnitude across all features.

Convergence

Stopping Criterion

||∇L|| → 0

Algorithm converges when the gradient magnitude approaches zero — minimum reached on the loss surface.

Section 03

Algorithm

Step-by-step gradient descent procedure from initialisation to convergence.

1

Define Problem & Collect Data

12 (area, price) pairs. Split into feature array X_raw and target Y.

2

Feature Normalisation

Apply z-score: X = (X_raw − μ) / σ. Stabilises gradient magnitudes.

3

Initialise Parameters

Set w = 0, b = 0. Choose α = 0.10, epochs = 500.

4

Forward Pass — Compute Predictions

ŷ = w·X + b for all m training samples.

5

Compute MSE Loss

L = (1/2m) · Σ(ŷ − y)². Record for convergence plot.

6

Backward Pass — Compute Gradients

∂L/∂w = (1/m)·dot(error, X), ∂L/∂b = (1/m)·sum(error).

7

Update Parameters

w ← w − α·dw, b ← b − α·db. One gradient step completed.

8

Repeat for All Epochs

Return to Step 4 until 500 epochs complete. Loss curve stored.

9

Evaluate & Visualise

Compute R², RMSE. Plot 6 visualisations: convergence, regression line, 3-D surface, contour, parameter traces, gradient magnitude.

Section 04

Code Implementation

Python implementation using NumPy. Core gradient descent engine shown below.

gradient_descent_project.py 
# ─── Dataset & Normalisation ──────────────────────────────────
import numpy as np

X_raw = np.array([650, 800, 1000, 1200, 1500, 1800,
                  2100, 2500, 3000, 3500, 4000, 4500])
Y     = np.array([45, 55, 65, 80, 95, 115, 135,
                   160, 195, 230, 265, 300])

X_mean, X_std = X_raw.mean(), X_raw.std()
X = (X_raw - X_mean) / X_std     # z-score normalisation
m = len(X)

# ─── Loss Function ────────────────────────────────────────────
def compute_loss(w, b, X, Y):
    predictions = w * X + b
    return (1 / (2 * m)) * np.sum((predictions - Y) ** 2)

# ─── Gradient Descent Engine ──────────────────────────────────
def gradient_descent(X, Y, w=0.0, b=0.0, lr=0.1, epochs=500):
    loss_hist = []
    for epoch in range(epochs):
        preds = w * X + b
        error = preds - Y

        dw = (1/m) * np.dot(error, X)   # ∂L/∂w  — gradient w.r.t. weight
        db = (1/m) * np.sum(error)       # ∂L/∂b  — gradient w.r.t. bias

        w -= lr * dw    # w ← w − α · ∂L/∂w
        b -= lr * db    # b ← b − α · ∂L/∂b

        loss_hist.append(compute_loss(w, b, X, Y))
    return w, b, loss_hist

# ─── Train ────────────────────────────────────────────────────
w_final, b_final, loss_history = gradient_descent(X, Y)

# ─── R² Score ─────────────────────────────────────────────────
Y_pred = w_final * X + b_final
ss_res = np.sum((Y - Y_pred) ** 2)
ss_tot = np.sum((Y - Y.mean()) ** 2)
r2     = 1 - ss_res / ss_tot
print(f"R² Score: {r2:.4f}")  # → 0.9989

Vector Calculus Concepts Applied

Every concept from the syllabus mapped to its role in this implementation.

∇

Gradient

∇L = [∂L/∂w, ∂L/∂b]

The gradient is the core engine of the optimiser — it points toward steepest loss increase, so we move opposite to it.

∂

Partial Derivatives

∂L/∂w and ∂L/∂b

Two partial derivatives form the gradient vector, one for each learnable parameter.

⛰

Scalar Field (Loss Surface)

L : ℝ² → ℝ

The MSE loss defines a scalar field over (w, b) space — visualised as a 3-D convex bowl.

→

Directional Derivative

D_u L = ∇L · û

The gradient descent direction −∇L / ||∇L|| is the unit direction of steepest descent.

∮

Convergence (Norm)

||∇L|| → 0

The Euclidean norm of the gradient vector approaches zero as the algorithm reaches a minimum.

⊕

Chain Rule

∂L/∂w = ∂L/∂ŷ · ∂ŷ/∂w

Backpropagation applies the multivariable chain rule to compute parameter gradients efficiently.
